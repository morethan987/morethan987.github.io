01 Background

    With the stable progress of LLM(Large Language Model) since the launch of ChatGPT by OpenAI, the imagination of the public has been greatly widened, which based on a simple fact: We can communicate with a machine now!

    Nevertheless, merely maintaining the level of "word solitaire" is not quite satisfactory: We wish the Artificial Intelligence presented by LLMs can be as intelligent as human.

    Or more practically, AI is supposed to act as an assistant for human scientist to push the boundary of Physical theory forward and not just a smart search engine with the ability to summary.

    So the first step is to gain a deeper insight about LLM, one of the most prevailing and successful AI format, with the considerable fame of Diffusion, another technic milestone of image generation.


02 Implementation

    By chance, I find an AMAZING GitHub project: Lightning-AI/litgpt which provides a chain of tools to grow a LLM including pretraining, finetuning, deploying and evaluating. Considering the scale of LLM models are stunning, I recommend install the repository in a python virtual environment.


    01 Virtual Environment

      1. Choose a disk with enough space and make a file folder to contain the litgpt, right-click and open in PowerShell and run this code:

      # 创建虚拟环境
python -m venv your_env_name

      Here are some useful parameter you can choose:
      --system-site-packages：当使用这个参数时，创建的虚拟环境将包含全局Python环境中的包。这可以避免重复安装一些常用的包。
      --clear：如果指定的虚拟环境目录已经存在，使用 '--clear' 会清除目录中的所有内容，然后重新创建虚拟环境。
      --help 或 '-h'：显示帮助信息，列出所有可用的参数和它们的说明。
      --version：显示 'venv' 模块的版本。
      --symlinks：在某些情况下，创建虚拟环境时会使用符号链接代替复制文件。这个参数可以控制这种行为。
      --upgrade：如果虚拟环境已经存在，使用 '--upgrade' 会升级其中的Python解释器到当前安装的Python版本。
      --with-pip：确保虚拟环境中包含pip工具。从Python 3.4开始，'pip' 已经是Python标准库的一部分，因此这个参数在较新版本的Python中不再需要。
      --debug：提供调试信息。
      Here is an example: 

      # 创建一个包含全局包的虚拟环境
python -m venv your_env_name --system-site-packages

      2. Activate the virtual environment: 

      # 激活虚拟环境
your_env_name/Scripts/activate

# 如果Power Shell版本需要可以添加'.\'在命令之前
.\your_env_name/Scripts/activate

    02 Download litgpt & pretrained LLM

      Now you can follow the official guidance: Zero to LitGPT to run LLM.

      1. When virtual environment is activated, run: 

      pip install 'litgpt[all]'

      2. When you are downloading the open source LLM, for example, through:
      litgpt download meta-llama\Meta-Llama-3.1-8B
      You are bounce to get an annoying error traceback:

      OSError: [WinError 126] 找不到指定的模块。 Error loading "E:\LitGPT\litgpt_env\Lib\site-packages\torch\lib\fbgemm.dll" or one of its dependencies.

      A missing dll file should be blamed for this issue and these haven't been an official solve. The easiest method to tackle this is to download that missing file, then simply place it in 'C:\Windows\System32'

    03 Run Pretrained LLM

      1. Through PowerShell.
      Activate the virtual environment, run: 

      litgpt chat microsoft/phi-2

      1. Through Python API.
      Here is a templet below: 

      from litgpt import LLM

llm = LLM.load("microsoft/phi-2") # 如果路径识别失败，改用：'llm = LLM.load("path/to/my/local/checkpoint")'
text = llm.generate("What do Llamas eat?", top_k=1, max_new_tokens=30)
print(text)
