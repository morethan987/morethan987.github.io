01 Precondition

--You are supposed to know how to download and correctly use a python interpreter, choose a suitable IDE and install package with pip .
--The main topics of this blog are python file operation and HTML element operation based on Beautifulsoup, so you'd better know something about them if you don't want to read "nonsense".
--This blog won't teach you to write a blog with AI, and the project are obviously imperfect: only suit my own website, can't be 100% automake, and strong dependence on direction of local file system......So don't take it seriously, just treat it as a project for a man new to programing.


02 Implementation

  1. Let's contemplate the essence of our goal. Crafting a new blog on my website presents a challenge: I must author HTML documents manually and navigate the intricate interconnections among various pages. And what's more, it's a completely repetitive and exhausting work, which is quite suitable for computer to deal with! What I need to do is to describe how I deal with that problems with CODE.
    To keep the style coherent, the attribute 'class' shouldn't be changed since the bundling relationship between 'class' and text style.
    Fortunately, I wrote a 'template' html document before: Local OverLeaf My program can simply copy some code from it.
    Theoretically feasible, let's begin the practice.

  2. In the early stage, we can simply consolidate all the code into a single file, which is easier to figure out the logic of our code with the program running step by step.
    During this period, you will gain a general understanding of the overall code manipulation: read a template file, locate the specific html element and modify its attribute, write your modified file.
    That's the core logic of my code: read -> locate -> modify -> write. Here are some typic example below. 
    
        # 解析模板文件
        print("开始编辑blogdetails，读取details模板中......")

        # 读取原始文件内容

        soup.find('h6', string='POSTED ON:')  # 标签内容定位

            file.write(str(soup))

    Honestly, all of these codes are so direct and basic that you can simply ask AI anything you don't understand.Let's just continue.

  3. Following initial coding endeavors, you may come across a moderately-sized issue: there is a need for information interchange among various pages, including the timestamp of your posts, the tags you append, and the name of your new file directories......
    Definitely you can announce a global variable to store the information and the following step and directly use it. However, you are overdrawing the ceiling of your program. A month later, while you staring at your code, struggling to guess what the strange variable name means, you will know what I am saying now.
    The easiest method to solve this problem is to divide a single code into several pieces with only one mission each piece need to do. But how can we get a graceful enough scheme to divide code into pieces ?
    Well, "Read extensively, and your write will flow as if guided by Muse", here is my solution.

            'detail_title': '',
            'category': '',
            'post_date': '',

        }  # 博客详情中的信息

        }  # 封面中的信息
        self.list_information = {}  # 列表中的信息
        self.tags_information = {}  # 标签中的信息

    def run(self):
        # 初始化检验文件夹创建
        create_folder()
        print("Program has create relative folders in 'D:/blog_writer'")

        # 撰写博客详情页
        details_writer = DetailsWriter(self)
        details_writer.start()

        # 撰写博客封面页
        pages_writer = PagesWriter(self)
        pages_writer.start()

        # 撰写博客列表页
        list_writer = ListWriter(self)
        list_writer.start()

        # 撰写博客标签页
        tags_writer = TagsWriter(self)
        tags_writer.start()

        # 结束
        end()
